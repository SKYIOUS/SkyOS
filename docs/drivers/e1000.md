# Intel e1000 Network Driver

The Intel e1000 driver provides networking for Intel Gigabit Ethernet adapters.

## Supported Hardware

- Intel 82540EM (desktop)
- Intel 82545EM (server)
- Intel 82546EB (dual port)
- Intel 82573E/L (mobile)

## Initialization

The driver initializes the NIC by:
1. Resetting the hardware and stopping DMA
2. Allocating transmit and receive descriptor rings (aligned to 16 bytes)
3. Configuring MAC address, multicast filters, and flow control
4. Setting up interrupt moderation (Interrupt Throttle Rate)
5. Enabling DMA and interrupts

```rust
pub struct E1000Driver {
    mmio: MmioRegion,
    tx_ring: DmaRing<TxDescriptor>,
    rx_ring: DmaRing<RxDescriptor>,
    mac_address: MacAddress,
    link_speed: u32,
}
```

## DMA Rings

Descriptors are organized in fixed-size circular rings. Each descriptor points to a data buffer:

```rust
#[repr(C, packed)]
pub struct TxDescriptor {
    pub address: PhysAddr,
    pub length: u16,
    pub cso: u8,
    pub cmd: u8,
    pub status: u8,
    pub css: u8,
    pub special: u16,
}
```

## Interrupt Handling

The driver uses MSI-X interrupts when available, falling back to pin-based interrupts. Interrupts are handled by reading the Interrupt Cause Read (ICR) register and processing the appropriate events (link up/down, transmit complete, receive available).

## Link Management

The driver monitors link status through the Status register and optionally through link-state change interrupts. Link speed (10/100/1000 Mbps) and duplex mode are auto-negotiated.
