# Project Roadmap

SkyOS development is organized into eight phases, each building on the previous.

## Development Phases

| Phase | Focus | Status |
|-------|-------|--------|
| Phase 1 | Kernel stabilization | In progress |
| Phase 2 | Networking improvements | Planned |
| Phase 3 | GUI and user experience | Planned |
| Phase 4 | Userspace ecosystem | Planned |
| Phase 5 | Driver support | Long-term |
| Phase 6 | Security features | Long-term |
| Phase 7 | Performance optimization | Long-term |
| Phase 8 | Portability | Stretch goal |

## Current Milestones

- **v0.1.0**: Bootable kernel with serial console, basic memory management, and scheduler
- **v0.2.0**: Filesystem support (tmpfs, ext2), syscall interface, and process management
- **v0.3.0**: Network stack (TCP/IP), socket API, and driver framework
- **v0.4.0**: GUI compositor, window management, and input handling
- **v0.5.0**: Userspace init system, package manager, and development tools

## Release Cadence

Major releases are targeted every 3-4 months. Patch releases are made as needed for bug fixes. The project follows semantic versioning for all releases.

## How to Help

See the [Contributing Guide](../guide/contributing.md) for ways to get involved. Priority areas are always marked with `help-wanted` labels on GitHub issues.
