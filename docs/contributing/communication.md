# Communication Channels

SkyOS contributors and users can connect through several communication channels. Each channel serves a different purpose.

## Discord

The [SkyOS Discord server](https://discord.gg/skyos) is the primary real-time communication platform. Use it for quick questions, design discussions, and casual conversation. Channels are organized by subsystem (e.g., `#kernel-memory`, `#drivers`, `#gui`) and project management (e.g., `#announcements`, `#sprints`). Be respectful and keep discussions on-topic. Off-topic conversation belongs in `#general`.

## GitHub Discussions

For longer-form technical discussions, feature proposals, and Q&A, use [GitHub Discussions](https://github.com/SkyOS/SkyOS/discussions). This is the right place to propose architectural changes before writing code, ask for help getting started, or share ideas that don't yet have a concrete implementation. Discussions are categorized as Ideas, Q&A, Show and Tell, and General.

## Mailing List

The SkyOS developer mailing list (`skyos-dev@lists.skyos.dev`) is used for asynchronous communication including patch series, RFCs, and periodic project updates. Subscription is open to anyone. This channel is lower-traffic than Discord and is archived publicly. Use the mailing list for formal proposals and patches that benefit from permanent record-keeping.

## Issue Tracker

Bug reports and feature requests go in the [GitHub issue tracker](https://github.com/SkyOS/SkyOS/issues). Search existing issues before filing a new one. Use the provided issue templates to ensure complete information. Security vulnerabilities should be reported privately (see `docs/security/vulnerability_reporting.md`) rather than through the public issue tracker.
